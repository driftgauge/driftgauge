# Security Policy

If you find a security issue, please avoid posting exploit details, credentials, or real user writing samples in a public issue.

## Preferred reporting
- Use GitHub private vulnerability reporting if it is enabled for the repository.
- Otherwise, contact the repository maintainers privately before opening a public issue.

## Please include
- affected endpoint or file
- reproduction steps
- impact
- any suggested fix

## Scope notes
This repository is a prototype and includes privacy-sensitive workflows. Please be especially careful not to include:
- real personal writing data
- access tokens or API keys
- private email addresses that are not already public project contacts
